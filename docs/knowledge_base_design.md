# 知识库与模型调校设计

## 1. 为什么需要知识库

Qwen 本身是通用模型，不知道你的机器人具体硬件、项目经历、用户偏好和功能边界。知识库的作用是把这些“项目私有信息”在提问时补充给模型。

## 2. 知识库内容建议

- 项目功能说明；
- 硬件组成：ESP32-S3、ESP32-C3、屏幕、摄像头、音频、IMU、小车底盘；
- 软件组成：Wi-Fi 配网、网页控制、AI 对话、TTS、摄像头、巡线；
- 用户画像：常问问题、学习目标、偏好的回答风格；
- 情感支持规则：先共情，再建议；
- 安全边界：不能执行危险动作，不能假装看到用户。

## 3. 从轻量检索升级到 RAG

当前示例使用关键词检索，适合面试演示。升级路线：

1. 文档切块；
2. 调用 embedding 模型生成向量；
3. 存入 FAISS / Chroma / Milvus；
4. 用户提问时检索 top-k；
5. 将检索结果与 system prompt 拼接；
6. 调用 Qwen 生成回答。

## 4. 模型调校不是训练

如果只是 prompt、知识库、参数和接口调整，应称为：

- prompt engineering；
- RAG 知识库增强；
- 模型应用层适配；
- 机器人交互策略设计。

只有使用数据集对模型权重进行 SFT/LoRA/微调，才叫训练或微调模型。
