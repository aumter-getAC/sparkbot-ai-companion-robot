# 履历写法建议

## 不建议写法

- 独立开发了完整 ESP-SparkBot 硬件系统；
- 从零训练了 Qwen 大模型；
- 完整复现并运行所有硬件功能。

这些说法容易被追问穿。

## 稳妥写法

项目名称：基于 ESP32-S3 的桌面陪伴机器人 AI 服务链路复现与优化

项目描述：

- 基于开源 SparkBot / sparkbot-45coll 平台，复现并梳理 ESP32-S3 头部主控、ESP32-C3 底盘、语音/屏幕/摄像头/网页控制等模块的软件交互链路；
- 完成 Qwen OpenAI-compatible API 接入，将原有 AI 对话链路改造为“用户输入—知识库检索—大模型生成—语音/屏幕输出—动作命令解析”的闭环；
- 设计桌面陪伴型 system prompt 和轻量知识库，支持项目问答、用户服务、情绪支持和机器人动作意图识别；
- 在无硬件条件下补充电脑端模拟器，模拟 ESP32 请求、AI 回复和底盘动作命令，便于调试和演示；
- 补充 Qwen LoRA/SFT 微调实验方案，构建桌面机器人问答与情绪支持样本，用于优化回答风格和动作控制边界。

## 简历 bullet 示例

- 负责桌面机器人 AI 软件链路改造，基于 Qwen API、Prompt Engineering 与轻量 RAG，实现个性化问答和情绪支持能力；
- 设计 AI Bridge 中间层，统一处理模型调用、知识库检索、上下文管理和自然语言动作命令解析，降低 ESP32 端资源占用；
- 构建机器人场景 SFT 数据集，并使用 LoRA 方案进行轻量微调实验，使模型回复更适合语音播报和硬件动作控制；
- 通过电脑端模拟器复现 ESP32 输入/输出协议，在无硬件条件下完成软件链路验证。

## 面试时一句话边界

> 我当时主要负责软件侧的大模型接入与机器人交互适配；目前由于没有硬件，我做的是软件链路复现与优化，并补充了 LoRA 微调实验和电脑端模拟器来支撑演示。
