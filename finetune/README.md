# Qwen 微调说明：如何把“微调”真实补进项目履历

> 重要边界：如果你当年没有做过微调，不建议在履历中写“当时完成了 Qwen 微调”。更稳妥的写法是：
> “在复现阶段补充完成了面向桌面机器人场景的 Qwen LoRA/SFT 微调实验，并与 RAG 方案进行对比。”

## 1. 微调、知识库、Prompt 的区别

- Prompt 调校：不改模型权重，只改 system prompt、temperature、max_tokens 等参数。
- 知识库/RAG：不改模型权重，把项目资料检索出来拼进上下文，让模型基于材料回答。
- 微调/SFT/LoRA：使用训练样本更新模型参数。LoRA 通常冻结原始基座模型权重，只训练额外的低秩适配器权重。

## 2. 本项目适合微调什么

不要微调“百科知识”，而是微调“机器人说话风格与任务格式”：

1. 回答更短，更适合语音播报；
2. 情绪支持时先共情，再给小建议；
3. 项目介绍时能稳定说出 ESP32-S3、ESP32-C3、Qwen、知识库、动作控制闭环；
4. 运动控制场景只做意图确认，不输出危险底层参数；
5. 输出更稳定，减少 Markdown、表格和长篇大论。

## 3. 数据格式

`data/sparkbot_sft_demo.json` 使用 LLaMA-Factory 支持的 alpaca 格式：

```json
{
  "instruction": "用户指令",
  "input": "可选输入",
  "output": "期望模型回答",
  "system": "系统提示词"
}
```

真实训练建议至少准备 100-300 条样本，分成：

- 项目问答类；
- 情绪支持类；
- 桌面助手服务类；
- 机器人动作控制类；
- 拒答和安全边界类。

## 4. 推荐训练方案

电脑没有独显时，建议用 Colab / AutoDL / 阿里云 GPU。轻量方案：

- 基座模型：Qwen2.5-0.5B-Instruct 或 Qwen2.5-1.5B-Instruct；
- 方法：LoRA 或 QLoRA；
- 框架：LLaMA-Factory；
- 输出：LoRA adapter 权重，不覆盖原始 Qwen 权重。

Qwen 官方训练文档给出了 LLaMA-Factory 的 SFT 流程，支持单卡/多卡，以及 full-parameter、LoRA、Q-LoRA、DoRA；自定义数据可以用 alpaca 或 sharegpt 格式。

## 5. 权重到底是什么

如果你做 LoRA：

```text
最终可部署模型 = Qwen 基座模型权重 + 你训练出来的 LoRA adapter 权重
```

- 基座权重：Qwen 官方开放模型参数，通常很大；
- adapter 权重：你的项目训练产生的小文件，体现“桌面机器人任务适配”；
- 合并权重：可以把 adapter 合并进基座模型，导出一个合并后的模型目录。

面试时说：

> 我采用参数高效微调 LoRA，而不是全量微调。这样可以冻结大部分基座模型权重，只训练少量适配器参数，在数据量和算力有限的情况下更适合学生项目。

## 6. 与 RAG 的关系

- 微调适合固化“回答风格、任务格式、动作边界”；
- RAG 适合补充“会变的知识、项目资料、用户偏好、说明书”。

本项目最合理的架构是：

```text
Qwen LoRA 微调模型：负责稳定的机器人语气和任务格式
RAG 知识库：负责项目资料、用户偏好、常见问答
AI Bridge：负责把两者接起来，并返回 reply + command
```
